## 1. Project Topology

### Workspace Structure
```
deriv-jules/
├── apps/                    # 5 tenant applications
│   ├── tenant-app/          # Primary tenant (base)
│   ├── tenant-app-1/        # Tenant variant 1
│   ├── tenant-app-2/        # Tenant variant 2
│   ├── tenant-app-3/        # Tenant variant 3
│   └── tenant-app-4/        # Tenant variant 4
├── libs/shared/             # 7 shared libraries (80% code)
│   ├── core-routes/         # Central routing & page exports
│   ├── ui/                  # Shared UI components
│   ├── theme/               # Tailwind v4 theming
│   ├── deriv-api/           # Deriv WebSocket API client
│   ├── supabase/            # Supabase client utilities
│   ├── shared-auth/         # Authentication logic
│   └── shared-config/       # Configuration utilities
├── e2e/                     # Playwright E2E tests
├── supabase/                # Database migrations
│   └── migrations/
│       └── 00001_create_rbac_schema.sql
└── docs/                    # Existing documentation
    ├── specifications.md    # Comprehensive spec (1636 lines)
    └── project-documentation.md
```

### Architecture Pattern
**80/20 Multi-Tenant Architecture:**
- 80% shared code in `libs/shared/*`
- 20% tenant-specific in `apps/tenant-app-*`
- Thin tenant apps consume shared libraries
